# CortexAI 多用户支持分析

## 结论
✅ **该平台完全支持多用户同时使用，且数据隔离良好**

## 用户隔离机制

### 1. 会话隔离

#### 数据库层面
- **表**: `ai_agent_session`
- **关键字段**:
  - `user_login_name`: 用户登录名（必填）
  - `business_system`: 业务系统标识（默认'cortex'）
  - `session_id`: 会话唯一ID（雪花算法）

#### 查询隔离
```java
// 前端请求
const res = await request({
  url: '/agent/api/session/list',
  params: {
    businessSystem: currentBusinessSystem.value,
    userLoginName: currentUser.value  // 当前登录用户
  }
})

// 后端查询
sessionManager.listSessions(businessSystem, userLoginName)
```

**结果**: 每个用户只能看到自己的会话列表

### 2. 文件隔离

#### 数据库层面
- **表**: `ai_agent_file`
- **关键字段**:
  - `agent_code`: Agent编码
  - `session_id`: 会话ID
  - `file_path`: 文件相对路径

#### 上传隔离
```java
// AgentFileController.uploadFile()
String agentCode = request.getParameter("agentCode");
// 文件存储在 /profile/agent/{year}/{month}/{day}/xxx
```

**结果**: 文件通过会话ID关联，只有会话所有者能访问

### 3. 技能包隔离（三层权限）

#### Skill权限模型
```java
// 三层隔离机制
1. 系统级技能: skill_scope='system' AND business_system='cortex'
2. 个人技能: skill_scope='user' AND owner_user='当前用户'
3. Agent级技能: 通过 agent_skill 关联表授权
```

#### 查询隔离
```sql
SELECT * FROM skill_node 
WHERE (skill_scope='system' AND business_system=#{businessSystem})
   OR (skill_scope='user' AND owner_user=#{userLoginName})
```

**结果**: 用户只能看到系统公共技能和自己的个人技能

### 4. 插件隔离

#### 当前状态
- 插件表 `ai_plugin` 没有用户字段
- **所有用户共享插件**
- 通过 Agent 配置来限制哪些插件可用

#### 权限控制
```java
// Agent -> Plugin 关联表
ai_agent_plugin (agent_id, plugin_id)
```

**结果**: 插件本身是全局共享的，但通过Agent配置实现访问控制

### 5. 消息存储隔离

#### Redis缓存层
```java
// SessionManager.java
private static final String SESSION_CTX_PREFIX = "agent:session:ctx:";
private static final String SESSION_MSG_PREFIX = "agent:session:msg:";

// Key格式
agent:session:ctx:{sessionId}
agent:session:msg:{sessionId}
```

#### 数据库层
```java
// MessageStore使用sessionId作为唯一标识
// 每个会话的消息独立存储
```

**结果**: 会话上下文和消息完全隔离，无法跨会话访问

## 并发支持

### 1. 会话锁机制
```java
// AgentSessionContext.java
private final ReentrantLock lock = new ReentrantLock();

public void lock() {
    lock.lock();
}

public void unlock() {
    lock.unlock();
}
```

**功能**: 
- 防止同一会话并发执行
- 支持不同会话并发执行
- 保证数据一致性

### 2. 雪花ID生成
```java
// SnowflakeIdWorker.java
// 分布式唯一ID生成器
// 支持多节点部署
```

**功能**:
- 保证会话ID全局唯一
- 支持分布式部署
- 高性能（每秒可生成400万个ID）

## 多租户支持

### Business System 概念
```java
// AiAgentSession.businessSystem
// 默认值: "cortex"
```

**用途**:
- 支持多个业务系统共用一个平台
- 每个业务系统的数据相互隔离
- 技能包可以按业务系统区分

**使用场景**:
- 企业A: businessSystem='company_a'
- 企业B: businessSystem='company_b'
- 内部使用: businessSystem='cortex'

## 部署建议

### 1. 单实例部署
- ✅ 支持多用户
- ✅ 数据隔离良好
- ⚠️ 会话锁基于JVM内存（单节点有效）

### 2. 集群部署
需要考虑：
- 使用Redis分布式锁替代ReentrantLock
- 共享Redis缓存
- 负载均衡配置

### 3. 容量规划
**单节点参考**:
- 100并发用户: 无压力
- 1000并发会话: 需要优化
- 建议: 每个节点50-200并发用户

## 潜在问题

### 1. 会话锁仅在单节点有效
```java
// 当前实现
private final ReentrantLock lock = new ReentrantLock();

// 集群部署需要改为
Redisson分布式锁
```

### 2. 插件全局共享
- 所有用户看到相同的插件列表
- 无法为特定用户启用/禁用插件
- 建议: 添加用户级插件开关

### 3. 文件存储路径
- 当前按时间分组: `/profile/agent/2024/01/15/xxx.pdf`
- 建议: 按用户分组: `/profile/agent/{user}/xxx.pdf`

## 总结

### ✅ 已支持
1. ✅ 用户登录认证（JWT）
2. ✅ 会话数据隔离（按用户）
3. ✅ 技能包权限隔离（三层）

6. ✅ 消息历史隔离
7. ✅ 文件上传隔离

### ⚠️ 需要注意
1. ⚠️ 集群部署需要分布式锁
2. ⚠️ 插件目前全局共享
3. ⚠️ 文件路径建议优化

### 🎯 结论
**该平台完全支持多用户同时使用**，数据隔离机制完善，适合企业级多用户场景。单节点部署可支持100+并发用户，集群部署需要少量改造（主要是分布式锁）。
