# 插件分类设计说明

## 📋 设计概述

将插件系统分为**内置插件**和**MCP插件**两大类，通过Tab分栏进行管理，提供清晰的用户体验。

---

## 🎯 核心设计理念

### **1. 内置插件（Built-in Plugins）**
- **用户可编辑**: 用户可以手动创建、编辑、删除
- **直接管理**: 在系统内部直接配置和管理
- **适用场景**: 
  - 自定义业务逻辑
  - 内部工具集成
  - 特定功能扩展

### **2. MCP插件（MCP Plugins）**
- **自动扫描**: 系统自动扫描已安装的Python/Node.js包
- **检测识别**: 自动识别MCP相关包（包含`mcp-server`、`@modelcontextprotocol`关键词）
- **配置驱动**: 用户只需填写配置参数，无需编写代码
- **适用场景**:
  - 使用已安装的MCP服务器包
  - 快速集成第三方MCP工具
  - 标准化的MCP协议实现

## 🏗️ 技术架构

### **后端实现**

#### 1. **数据模型（VO）**

```java
// McpPackageInfo.java - MCP包信息
public class McpPackageInfo {
    private String packageName;       // 包名
    private String displayName;       // 显示名称
    private String version;           // 版本号
    private String description;       // 描述
    private String runtimeType;       // 运行时类型（uvx/npx）
    private Boolean configured;       // 是否已配置
    private Long pluginId;            // 插件ID（如果已配置）
    private String defaultCommand;    // 默认启动命令
    private String author;            // 作者
    private String category;          // 分类
    private Boolean official;         // 是否官方推荐
}

// McpPackageScanResult.java - 扫描结果
public class McpPackageScanResult {
    private List<McpPackageInfo> pythonPackages;  // Python包列表
    private List<McpPackageInfo> nodePackages;    // Node.js包列表
    private Integer totalCount;                   // 总计
    private Integer configuredCount;              // 已配置
    private Integer unconfiguredCount;            // 未配置
    private String scanTime;                      // 扫描时间
    private Boolean pythonAvailable;              // Python环境可用性
    private Boolean nodeAvailable;                // Node环境可用性
}
```

#### 2. **服务层接口**

```java
public interface IMcpPackageScanService {
    // 扫描所有MCP包
    McpPackageScanResult scanAllMcpPackages();
    
    // 扫描Python MCP包
    List<McpPackageInfo> scanPythonMcpPackages();
    
    // 扫描Node.js MCP包
    List<McpPackageInfo> scanNodeMcpPackages();
    
    // 自动配置插件
    Long autoConfigurePlugin(String packageName, String runtimeType);
    
    // 获取包元数据
    McpPackageInfo getPackageMetadata(String packageName, String runtimeType);
}
```

#### 3. **核心扫描逻辑**

**Python包扫描**:
```bash
pip list | grep mcp
```
- 执行`pip list`命令
- 筛选包含MCP关键词的包
- 解析包名和版本号
- 查询是否已在`ai_plugin`表中配置

**Node.js包扫描**:
```bash
npm list -g --depth=0 --json | grep @modelcontextprotocol
```
- 执行`npm list`命令
- 解析JSON输出
- 筛选MCP相关包
- 查询配置状态

#### 4. **官方包元数据**

系统内置官方MCP包的元数据：

| 包名 | 显示名称 | 分类 | 运行时 |
|------|---------|------|--------|
| mcp-server-sqlite | SQLite数据库 | database | uvx |
| mcp-server-postgres | PostgreSQL数据库 | database | uvx |
| @modelcontextprotocol/server-filesystem | 文件系统 | file_system | npx |
| @modelcontextprotocol/server-git | Git版本控制 | utility | npx |
| @modelcontextprotocol/server-brave-search | Brave搜索 | web_search | npx |

#### 5. **Controller接口**

```java
// 扫描所有MCP包
GET /plugin/list/scanMcpPackages

// 扫描Python包
GET /plugin/list/scanPythonPackages

// 扫描Node.js包
GET /plugin/list/scanNodePackages

// 自动配置MCP插件
POST /plugin/list/autoConfigureMcp

// 获取包元数据
GET /plugin/list/getPackageMetadata/{packageName}/{runtimeType}
```

## 🔄 用户工作流程

### **场景1: 使用已安装的MCP包**

1. 用户通过pip或npm全局安装MCP包
   ```bash
   pip install mcp-server-sqlite
   # 或
   npm install -g @modelcontextprotocol/server-filesystem
   ```

2. 进入【插件管理】→【MCP插件】Tab

3. 点击【扫描MCP包】按钮

4. 系统自动检测并显示已安装的MCP包

5. 找到目标包，点击【配置】按钮

6. 系统自动生成插件配置：
   - 插件名称（自动生成）
   - 显示名称（从元数据获取）
   - 启动命令（自动生成）
   - 分类、描述等信息

7. 点击【确认配置】，插件添加到系统

8. 切换到【内置插件】Tab，可查看和管理已配置的插件

### **场景2: 创建内置插件**

1. 进入【插件管理】→【内置插件】Tab

2. 点击【新增插件】按钮

3. 填写插件信息：
   - 插件名称
   - 插件名称
   - 插件类型（选择"内置"）
   - 自定义配置...

4. 保存插件配置

## 🔧 配置说明

### **MCP关键词识别规则**

系统会扫描包名中包含以下关键词的包：
- `mcp-server`
- `mcp-`
- `@modelcontextprotocol`

### **默认运行时选择**

| 包管理器 | 运行时类型 | 说明 |
|---------|-----------|------|
| pip | uvx | 推荐使用uvx自动管理Python环境 |
| npm | npx | 使用npx执行Node.js包 |

### **启动命令生成规则**

- **Python包**: `uvx <package-name>`
- **Node.js包**: `npx -y <package-name>`
