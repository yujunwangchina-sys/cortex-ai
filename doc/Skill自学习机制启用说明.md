# Skill 自学习机制启用说明

**日期**: 2026-07-03  
**问题**: Agent 从未主动创建个人技能  
**原因**: 系统有工具但没有引导语  
**解决方案**: 在系统提示词中添加学习指南

---

## 🔍 问题诊断

### 原有状态

✅ **已实现的技术能力**：
- `skill_create` 工具 - AI 可以创建技能
- `skill_edit` 工具 - AI 可以编辑技能
- `skills_list` 工具 - AI 可以查看技能
- `skill_view` 工具 - AI 可以查看技能内容
- 个人技能包自动创建机制
- 两层权限隔离（global + personal）

❌ **缺失的关键部分**：
- **没有任何提示词告诉AI应该使用这些工具**
- AI 不知道什么时候应该学习
- AI 不知道什么内容值得保存为技能

### 现象

用户文档（`Skill渐进式披露机制详解.md`）声称：
> "个人Skill由Agent自学习自动生成，无需手动分配"

但实际上：
- 从未见过 AI 主动调用 `skill_create`
- 个人技能包一直是空的
- 所有技能都需要人工在前端创建

**根本原因**：就像给了AI一支笔和纸，但从未告诉它"遇到新知识要记下来"。

---

## ✅ 解决方案

### 修改文件

`cortex-system/src/main/java/com/cortex/agent/runtime/prompt/PromptBuilder.java`

### 添加的内容

在系统提示词中添加了 **技能学习指南**（`SKILL_LEARNING_GUIDE`），告诉AI：

#### 1. 适合创建技能的场景
- 复杂问题的解决方案
- 特定领域的操作流程
- 代码模板和示例
- 故障排查指南
- 配置说明
- 用户明确要求记住

#### 2. 不适合创建技能的场景
- 简单的一次性回答
- 纯粹的聊天对话
- 已存在类似技能

#### 3. 创建技能的时机
- **在完成任务后**：主动询问用户
- **用户要求时**：响应"记住"、"保存"等指令
- **发现知识空白时**：填补技能库空白

#### 4. SKILL.md 格式要求
提供了标准模板，包括：
- YAML frontmatter (name, description, version, author, keywords)
- 适用场景
- 操作步骤
- 注意事项
- 示例代码

### 代码变更

```java
// 新增常量
private static final String SKILL_LEARNING_GUIDE =
    "\n## 💡 技能学习机制\n\n" +
    "当你在对话中遇到以下情况时，应该主动使用 **skill_create** 工具保存为个人技能：\n\n" +
    // ... (详细指南)

// 在 buildSystemPrompt 方法中注入
sb.append(SKILL_INDEX_HINT);
sb.append(SKILL_LEARNING_GUIDE);  // 👈 新增这一行
```

## 🧪 测试场景

### 场景1：复杂配置问题

**用户**：我的 Nginx 反向代理总是超时，怎么配置？

**预期**：
1. AI 给出详细配置方案
2. AI 主动询问："这个 Nginx 配置方案将来可能用得上，要不要保存为个人技能？"
3. 用户同意后，创建 `Nginx反向代理超时配置` 技能

### 场景2：故障排查

**用户**：Spring Boot 应用启动失败，端口被占用

**预期**：
1. AI 给出排查步骤和解决方法
2. AI 询问："这个端口占用排查流程可以保存下来，下次遇到类似问题就快多了。要保存吗？"
3. 创建 `Spring Boot端口占用排查指南` 技能

### 场景3：代码模板

**用户**：写一个 Java 的文件上传接口

**预期**：
1. AI 提供完整代码
2. AI 询问："这个文件上传模板挺通用的，要不要保存起来？以后新项目就能直接用。"
3. 创建 `Java文件上传接口模板` 技能

### 场景4：用户明确要求

**用户**：记住这个 Docker Compose 配置，以后经常用

**预期**：
1. AI 立即调用 skill_create
2. 不需要询问，因为用户明确说了"记住"
3. 创建对应技能

## 🔧 后续优化建议

### 1. 智能去重

在创建前调用 `skills_list` 检查是否已有类似技能：

```java
// 伪代码
if (skills_list 中已有 "Docker网络配置") {
    return "已有类似技能，要不要更新现有技能？";
} else {
    skill_create(...);
}
```

### 2. 自动命名

AI 根据内容自动生成技能名称，而不是让用户命名：

```
bad: "技能1"、"我的笔记"
good: "Nginx反向代理超时配置"、"Docker网络桥接配置"
```

### 3. 技能质量评估

记录技能的使用频率，低质量技能提示用户删除：

```sql
ALTER TABLE skill_node ADD COLUMN usage_count INT DEFAULT 0;
ALTER TABLE skill_node ADD COLUMN last_used_time DATETIME;
```

### 4. 技能合并

发现多个相似技能时，提示合并：

```
发现 3 个关于 Docker 的技能，要不要合并为"Docker运维指南"？
```

### 5. 技能升级

优秀的个人技能提升为全局技能：

```
您的"MySQL慢查询优化"技能被使用了 20 次，
管理员可以将其提升为全局技能，供所有用户使用。
```

## ⚠️ 注意事项

### 1. Token 消耗

每次对话都会注入学习指南（约 800 tokens），需权衡：
- **优势**：AI 随时知道该学习
- **劣势**：每次对话都消耗 token

**优化方案**：
- 只在检测到"复杂问题解决"后注入
- 或者用更简洁的提示语（200 tokens）

### 2. 用户体验

AI 频繁询问"要不要保存"可能打扰用户：

**解决方案**：
- 添加用户偏好设置："自动保存"、"总是询问"、"从不保存"
- 只在真正有价值的内容时询问（AI 自己判断阈值）

### 3. 技能质量

AI 可能保存低质量内容：

**解决方案**：
- 提供"技能评分"功能
- 用户可以删除或标记"无用"
- AI 学习什么内容值得保存
