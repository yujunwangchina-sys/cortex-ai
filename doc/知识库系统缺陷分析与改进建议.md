### 4. 知识库检索结果合并逻辑简单
**问题描述**：
- 多个知识库结果可能重复，没有去重
- 固定取 top 10，不够灵活
- 没有记录结果来源的知识库信息

**影响**：低 - 影响检索质量

**建议改进**：
```java
public List<SearchResult> searchForAgent(Long agentId, String query, String retrievalMode, Integer maxResults) {
    List<AiAgentKnowledge> grants = agentKbMapper.selectActiveByAgentId(agentId);
    if (grants == null || grants.isEmpty()) {
        return Collections.emptyList();
    }

    List<SearchResult> allResults = new ArrayList<>();
    
    for (AiAgentKnowledge grant : grants) {
        if (retrievalMode != null && !retrievalMode.equals(grant.getRetrievalMode())) {
            continue;
        }
        
        List<SearchResult> kbResults = search(
                grant.getKbId(), query, null, null, grant.getMetadataFilter());
        
        // 标记来源知识库
        for (SearchResult sr : kbResults) {
            sr.setKnowledgeBaseId(grant.getKbId());
            sr.setKnowledgeBaseName(grant.getKbName());
        }
        
        allResults.addAll(kbResults);
    }

    // 去重：基于content的hash
    Map<String, SearchResult> uniqueResults = new HashMap<>();
    for (SearchResult sr : allResults) {
        String contentHash = String.valueOf(sr.getContent().hashCode());
        SearchResult existing = uniqueResults.get(contentHash);
        if (existing == null || sr.getScore().compareTo(existing.getScore()) > 0) {
            uniqueResults.put(contentHash, sr);
        }
    }

    // 排序并限制数量
    List<SearchResult> finalResults = new ArrayList<>(uniqueResults.values());
    finalResults.sort((a, b) -> b.getScore().compareTo(a.getScore()));
    
    int limit = maxResults != null ? maxResults : 10;
    if (finalResults.size() > limit) {
        return finalResults.subList(0, limit);
    }

    return finalResults;
}
```

## 💡 性能优化建议

### 9. 缓存优化
**当前问题**：
- EmbeddingModel 和 EmbeddingStore 缓存没有过期机制
- 缓存无限增长可能导致内存问题

**建议**：
- 使用 Caffeine 或 Guava Cache 替代 ConcurrentHashMap
- 设置合理的过期时间（如 1 小时）
- 添加 LRU 淘汰策略

### 10. 批量嵌入优化
**当前问题**：每次检索都要调用 Embedding API，浪费资源

**建议**：
- 对于常见查询，缓存其 embedding 向量
- 支持批量 embedding（一次性处理多个查询）

### 11. 检索结果缓存
**当前问题**：相同查询重复检索浪费资源

**建议**：
- 使用 Redis 缓存热门查询的检索结果
- 设置合理的 TTL（如 15 分钟）

## 🔒 安全建议

### 12. 知识库访问权限检查

- 脱敏处理（如手机号、身份证）
