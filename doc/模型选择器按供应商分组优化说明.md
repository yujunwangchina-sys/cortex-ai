# 模型选择器按供应商分组优化说明

## 修改时间
2026-07-06

## 需求描述
原来的模型选择器直接显示模型名称，不够清晰。优化为：
- 按钮显示当前选中模型的**供应商名称**
- 鼠标悬浮时展开下拉菜单
- 菜单按**供应商分组**
- 悬浮在供应商名称上时，右侧展开该供应商的所有模型

## 实现方案

### 前端修改

**文件**：`ChatInputBox.vue`

#### 1. 修改触发方式
```vue
<!-- 从click改为hover触发 -->
<el-dropdown trigger="hover" placement="top" @command="handleModelChange">
```

#### 2. 修改按钮显示
```vue

<el-button link class="toolbar-btn model-btn">
  <el-icon><Setting /></el-icon>
  <span class="btn-text">{{ supplierLabel }}</span>
  <el-icon class="arrow-icon"><ArrowDown /></el-icon>
</el-button>
```

#### 3. 嵌套下拉菜单结构
```vue
<template #dropdown>
  <el-dropdown-menu class="model-dropdown-menu">
    <!-- 第一层：供应商列表 -->
    <template v-for="supplier in modelsBySupplier" :key="supplier.supplierName">
      <el-dropdown trigger="hover" placement="right-start" :hide-on-click="false">
        <el-dropdown-item class="supplier-item">
          {{ supplier.supplierName }}
          <el-icon class="supplier-arrow"><ArrowRight /></el-icon>
        </el-dropdown-item>
        
        <!-- 第二层：该供应商的模型列表 -->
        <template #dropdown>
          <el-dropdown-menu class="model-submenu">
            <el-dropdown-item
              v-for="m in supplier.models"
              :key="m.modelId"
              :command="m.modelId"
              :class="{ 'is-active': m.modelId === selectedModelId }"
            >
              <el-icon v-if="m.modelId === selectedModelId" class="check-icon">
                <CircleCheck />
              </el-icon>
              {{ m.modelName }}
            </el-dropdown-item>
          </el-dropdown-menu>
        </template>
      </el-dropdown>
    </template>
  </el-dropdown-menu>
</template>
```

#### 4. 添加计算属性

```javascript
// 当前选中模型的供应商名称
const supplierLabel = computed(() => {
  const model = props.models.find(m => m.modelId === props.selectedModelId)
  return model && model.supplierName ? model.supplierName : '选择供应商'
})

// 按供应商分组的模型列表
const modelsBySupplier = computed(() => {
  const groups = {}
  
  props.models.forEach(model => {
    const supplierName = model.supplierName || '未知供应商'
    if (!groups[supplierName]) {
      groups[supplierName] = {
        supplierName,
        models: []
      }
    }
    groups[supplierName].models.push(model)
  })
  
  // 转换为数组并排序
  return Object.values(groups).sort((a, b) => {
    return a.supplierName.localeCompare(b.supplierName, 'zh-CN')
  })
})
```

## 用户体验优化

### 优势
1. ✅ **更清晰的视觉层次**：按供应商分组，一目了然
2. ✅ **简洁的按钮显示**：显示供应商名称，节省空间
3. ✅ **悬浮触发**：移动端也可以快速查看
4. ✅ **选中状态明确**：当前模型高亮+对号图标
5. ✅ **自动排序**：供应商按拼音排序，模型按添加顺序

### 交互流程
1. 用户看到按钮显示"OpenAI"（当前供应商）
2. 鼠标悬浮在按钮上 → 展开供应商列表
3. 鼠标移到某个供应商（如"DeepSeek"）→ 右侧展开该供应商的模型
4. 点击具体模型（如"DeepSeek-V3"）→ 切换模型
5. 按钮文字自动更新为"DeepSeek"

## 数据依赖

### 后端API：`/agent/api/models`
返回的模型数据需要包含 `supplierName` 字段：

```json
[
  {
    "modelId": 1,
    "modelName": "GPT-4",
    "modelCode": "gpt-4",
    "supplierId": 1,
    "supplierName": "OpenAI",  // ← 必需字段
    "modelType": "chat",
    "contextLength": 128000,
    "maxTokens": 16384,
    ...
  }
]
```

### 数据库关联
`ai_model` 表通过 `supplierId` 关联 `ai_supplier` 表：
- Mapper XML 中使用 `left join` 关联查询
- `supplierName` 作为关联字段返回

## 兼容性

- ✅ **向后兼容**：如果某个模型没有 `supplierName`，会归类到"未知供应商"
- ✅ **响应式适配**：移动端隐藏按钮文字，只显示图标
- ✅ **多供应商支持**：自动适配任意数量的供应商和模型

## 测试场景

### 场景1：单个供应商
- 只有OpenAI的模型
- 显示：OpenAI → GPT-4, GPT-3.5-Turbo

### 场景2：多个供应商
- OpenAI, DeepSeek, Qwen
- 显示三个供应商，每个展开显示各自模型

### 场景3：供应商名称缺失
- 某个模型的supplierName为空
- 归类到"未知供应商"

### 场景4：移动端
- 按钮只显示图标⚙️
- 悬浮菜单正常工作
