# 时区问题修复说明

## 问题描述

系统显示的时间比实际时间晚8小时，例如：
- 实际创建时间：2026-07-06 15:00:00
- 显示时间：2026-07-06 07:00:00

这是典型的UTC时区和中国时区(UTC+8)的差异问题。

## 根本原因

1. **数据库存储**：MySQL存储的是本地时间（GMT+8）
2. **Jackson序列化**：`@JsonFormat`注解缺少`timezone`参数，默认使用UTC时区
3. **结果**：数据从数据库读取后，Jackson将其当作UTC时间进行格式化，导致显示时比实际时间少8小时

## 修复方案

### 1. BaseEntity时间字段修复

**文件**：`ruoyi-common/src/main/java/com/ruoyi/common/core/domain/BaseEntity.java`

**修改前**：
```java
/** 创建时间 */
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
private Date createTime;

/** 更新时间 */
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
private Date updateTime;
```

**修改后**：
```java
/** 创建时间 */
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private Date createTime;

/** 更新时间 */
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private Date updateTime;
```

### 2. AiAgentSession时间字段修复

**文件**：`ruoyi-system/src/main/java/com/ruoyi/agent/domain/AiAgentSession.java`

**修改前**：
```java
/** 最后消息时间 */
private java.util.Date lastMessageTime;
```

**修改后**：
```java
/** 最后消息时间 */
@com.fasterxml.jackson.annotation.JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private java.util.Date lastMessageTime;
```

## 已配置的时区设置

### 1. Spring Jackson配置

```yaml

```

## 注意事项

1. **已有数据不受影响**：数据库中的时间数据本身是正确的，只是显示有问题
2. **需要重启应用**：修改Java代码后需要重启后端服务才能生效
3. **前端无需修改**：前端直接显示后端返回的格式化时间字符串
4. **全局影响**：`BaseEntity`的修改会影响所有实体类

## 其他时区相关配置

如果需要支持其他时区，可以修改以下配置：

### 方式1：全局配置（推荐用于单时区应用）
```yaml
# application.yml
spring:
  jackson:
    time-zone: GMT+8  # 或其他时区如 Asia/Shanghai
```

### 方式2：注解配置（推荐用于多时区应用）
```java
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private Date createTime;
```

### 方式3：程序化配置
```java
@Configuration
public class JacksonConfig {
    @Bean
    public Jackson2ObjectMapperBuilderCustomizer jacksonObjectMapperCustomization() {
        return jacksonObjectMapperBuilder -> 
            jacksonObjectMapperBuilder.timeZone(TimeZone.getTimeZone("GMT+8"));
    }
}
```
