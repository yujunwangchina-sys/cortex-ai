# 内置插件开发规范

## 一、概述

内置插件是直接编译到系统中的Java插件，无需外部运行时环境，性能更好，集成更紧密。

## 二、核心接口

### 1. IBuiltinPlugin 接口

所有内置插件必须实现此接口：

```java
public interface IBuiltinPlugin {
    // 获取插件信息（必须）
    PluginInfo getPluginInfo();
    
    // 获取工具列表（必须）
    List<ToolDefinition> getTools();
    
    // 执行工具（必须）
    String executeTool(String toolName, Map<String, Object> arguments);
    
    // 初始化（可选）
    default void initialize() {}
    
    // 销毁（可选）
    default void destroy() {}
    
    // 检查可用性（可选）
    default boolean isAvailable() { return true; }
    
    // 获取状态（可选）
    default Map<String, Object> getStatus() {}
}
```

## 三、核心类

### 1. PluginInfo - 插件元数据

```java
PluginInfo info = new PluginInfo(
    "插件名称",
    "plugin-code",  // 唯一标识
    "插件描述"
);
info.setVersion("1.0.0");
info.setAuthor("作者");
info.setCategory("database");  // database/file_system/web_search/utility/custom
info.setEmoji("📚");
info.setRequireApproval(false);
```

### 2. ToolDefinition - 工具定义

```java
ToolDefinition tool = new ToolDefinition();
tool.setName("tool_name");
tool.setDescription("工具描述");

// 输入参数Schema（JSON Schema格式）
Map<String, Object> inputSchema = new HashMap<>();
inputSchema.put("type", "object");
inputSchema.put("properties", Map.of(
    "param1", Map.of(
        "type", "string",
        "description", "参数描述"
    )
));
inputSchema.put("required", List.of("param1"));
tool.setInputSchema(inputSchema);

// 示例
tool.setExampleInput("{\"param1\": \"value\"}");
tool.setExampleOutput("{\"success\": true, \"data\": {}}");
```

### 3. ToolResult - 执行结果

```java
// 成功结果
return ToolResult.success("操作成功")
    .addData("key1", value1)
    .addData("key2", value2)
    .toJson();

// 失败结果
return ToolResult.error("错误信息").toJson();
```

## 四、开发步骤

### 步骤1：创建插件类

在 `com.ruoyi.plugin.builtin.impl` 包下创建插件类：

```java
@Component  // 注册为Spring Bean
public class MyPlugin implements IBuiltinPlugin {
    
    private static final Logger log = LoggerFactory.getLogger(MyPlugin.class);
    
    // 可以注入Spring Bean和配置
    @Value("${my.config:}")
    private String myConfig;
    
    @Autowired(required = false)
    private SomeService someService;
}
```

### 步骤2：实现getPluginInfo()

```java
@Override
public PluginInfo getPluginInfo() {
    PluginInfo info = new PluginInfo(
        "我的插件",
        "my-plugin",
        "这是一个示例插件"
    );
    info.setVersion("1.0.0");
    info.setAuthor("开发者");
    info.setCategory("utility");
    info.setEmoji("🔧");
    return info;
}
```

### 步骤3：实现getTools()

```java
@Override
public List<ToolDefinition> getTools() {
    List<ToolDefinition> tools = new ArrayList<>();
    
    // 工具1
    ToolDefinition tool1 = new ToolDefinition();
    tool1.setName("my_tool");
    tool1.setDescription("工具描述");
    tool1.setInputSchema(createInputSchema());
    tools.add(tool1);
    
    return tools;
}

private Map<String, Object> createInputSchema() {
    return Map.of(
        "type", "object",
        "properties", Map.of(
            "query", Map.of(
                "type", "string",
                "description", "查询内容"
            )
        ),
        "required", List.of("query")
    );
}
```

### 步骤4：实现executeTool()

```java
@Override
public String executeTool(String toolName, Map<String, Object> arguments) {
    try {
        switch (toolName) {
            case "my_tool":
                return executeMyTool(arguments);
            default:
                return ToolResult.error("未知工具: " + toolName).toJson();
        }
    } catch (Exception e) {
        log.error("工具执行失败: " + toolName, e);
        return ToolResult.error("执行失败: " + e.getMessage()).toJson();
    }
}

private String executeMyTool(Map<String, Object> args) {
    // 1. 参数验证
    String query = (String) args.get("query");
    if (query == null || query.isEmpty()) {
        return ToolResult.error("query参数不能为空").toJson();
    }
    
    // 2. 执行业务逻辑
    try {
        String result = doSomething(query);
        
        // 3. 返回结果
        return ToolResult.success("执行成功")
            .addData("result", result)
            .toJson();
    } catch (Exception e) {
        return ToolResult.error("处理失败: " + e.getMessage()).toJson();
    }
}
```

### 步骤5：可选生命周期方法

```java
@Override
public void initialize() {
    // 插件加载时调用，可以初始化资源
    log.info("插件初始化");
}

@Override
public void destroy() {
    // 插件卸载时调用，可以释放资源
    log.info("插件销毁");
}

@Override
public boolean isAvailable() {
    // 检查插件是否可用（比如检查配置是否完整）
    return myConfig != null && !myConfig.isEmpty();
}
```

## 五、最佳实践

### 1. 配置管理

使用Spring的`@Value`注入配置：

```java
@Value("${plugin.api.url:}")
private String apiUrl;

@Value("${plugin.api.key:}")
private String apiKey;

@Value("${plugin.timeout:30}")
private int timeout;
```

在`application.yml`中配置：

```yaml
plugin:
  api:
    url: http://api-host/v1
    key: your-api-key
  timeout: 30
```

### 2. 依赖注入

可以注入Spring管理的Bean：

```java
@Autowired
private RestTemplate restTemplate;

@Autowired(required = false)  // 可选依赖
private RedisTemplate<String, Object> redisTemplate;
```

### 3. 错误处理

```java
try {
    // 业务逻辑
} catch (IllegalArgumentException e) {
    // 参数错误
    return ToolResult.error("参数错误: " + e.getMessage()).toJson();
} catch (IOException e) {
    // IO错误
    log.error("网络请求失败", e);
    return ToolResult.error("网络请求失败，请稍后重试").toJson();
} catch (Exception e) {
    // 未知错误
    log.error("未知错误", e);
    return ToolResult.error("系统错误: " + e.getMessage()).toJson();
}
```

### 4. 日志记录

```java
private static final Logger log = LoggerFactory.getLogger(MyPlugin.class);

log.info("插件初始化完成");
log.debug("执行工具: {} 参数: {}", toolName, arguments);
log.warn("配置缺失: {}", configKey);
log.error("执行失败", exception);
```

### 5. 异步处理

对于耗时操作，建议使用异步：

```java
@Autowired
private ThreadPoolTaskExecutor taskExecutor;

private String asyncOperation(Map<String, Object> args) {
    CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
        // 耗时操作
        return doSomething();
    }, taskExecutor);
    
    try {
        String result = future.get(30, TimeUnit.SECONDS);
        return ToolResult.success("成功").addData("result", result).toJson();
    } catch (TimeoutException e) {
        return ToolResult.error("操作超时").toJson();
    } catch (Exception e) {
        return ToolResult.error("操作失败: " + e.getMessage()).toJson();
    }
}
```

### 6. 缓存使用

```java
@Autowired(required = false)
private CacheManager cacheManager;

private String getCachedData(String key) {
    Cache cache = cacheManager.getCache("plugin-cache");
    if (cache != null) {
        String cached = cache.get(key, String.class);
        if (cached != null) {
            return cached;
        }
    }
    
    String data = fetchData(key);
    
    if (cache != null) {
        cache.put(key, data);
    }
    
    return data;
}
```

## 十、注意事项

1. **线程安全**：插件实例可能被多线程并发调用，注意线程安全
2. **资源释放**：在`destroy()`中释放资源（连接、线程池等）
3. **异常处理**：捕获所有异常，避免影响系统稳定性
4. **性能优化**：避免阻塞操作，使用连接池、缓存等优化性能
5. **安全性**：验证输入参数，防止注入攻击
6. **兼容性**：保持API向后兼容，版本升级时注意兼容性
