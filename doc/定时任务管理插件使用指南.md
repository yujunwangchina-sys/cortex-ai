# 定时任务管理插件使用指南

## 插件概述

**插件名称**：定时任务管理（Scheduled Task）  
**插件代码**：`scheduled-task`  
**版本**：1.0.0  
**类名**：`com.ruoyi.plugin.builtin.impl.ScheduledTaskPlugin`

定时任务管理插件允许AI Agent创建、查询、修改、删除和执行定时任务，基于Quartz调度框架实现。

---

## 插件加载

### 方式1：系统自动扫描
系统启动时会自动扫描并加载所有内置插件。

### 方式2：手动加载
1. 进入插件管理页面
2. 点击"扫描内置插件"
3. 找到"定时任务管理"插件并加载

## 工具列表

### 1. task_list - 查询任务列表

**描述**：查询系统中的定时任务列表，支持按名称和状态筛选。

**参数**：
- `jobName`（可选，string）：任务名称（模糊搜索）
- `status`（可选，string）：任务状态（0=正常，1=暂停）

**示例**：
```json
{
  "jobName": "报表",
  "status": "0"
}
```

**返回示例**：
```json
{
  "success": true,
  "message": "查询成功，共 2 个任务",
  "data": {
    "tasks": [
      {
        "jobId": 1,
        "jobName": "每日报表生成",
        "jobGroup": "REPORT",
        "invokeTarget": "ryTask.ryParams('report')",
        "cronExpression": "0 0 2 * * ?",
        "status": "正常",
        "concurrent": "禁止",
        "nextExecution": "2026-07-07 02:00:00",
        "remark": "每天凌晨2点生成报表",
        "createTime": "2026-07-06 10:00:00"
      }
    ],
    "total": 2
  }
}
```

### 2. task_create - 创建定时任务

**描述**：创建新的定时任务。

**必填参数**：
- `jobName`（string）：任务名称（如：每日报表生成）
- `jobGroup`（string）：任务组名（如：DEFAULT、REPORT、BACKUP）
- `invokeTarget`（string）：调用目标字符串（如：`ryTask.ryNoParams`）
- `cronExpression`（string）：Cron表达式

**可选参数**：
- `misfirePolicy`（string）：错过执行策略
  - `0`：默认策略
  - `1`：立即触发执行
  - `2`：触发一次执行
  - `3`：不触发立即执行
- `concurrent`（string）：是否允许并发
  - `0`：允许并发
  - `1`：禁止并发（默认）
- `remark`（string）：备注说明

**Cron表达式常用示例**：
- 每分钟执行：`0 * * * * ?`
- 每小时执行：`0 0 * * * ?`
- 每天凌晨2点：`0 0 2 * * ?`
- 每周一9点：`0 0 9 ? * MON`
- 每月1号凌晨：`0 0 0 1 * ?`
- 每个工作日9点：`0 0 9 ? * MON-FRI`

**示例**：
```json
{
  "jobName": "每日备份任务",
  "jobGroup": "BACKUP",
  "invokeTarget": "ryTask.ryParams('backup')",
  "cronExpression": "0 0 3 * * ?",
  "misfirePolicy": "0",
  "concurrent": "1",
  "remark": "每天凌晨3点执行数据库备份"
}
```

**返回示例**：
```json
{
  "success": true,
  "message": "定时任务创建成功",
  "data": {
    "jobId": 5,
    "jobName": "每日备份任务",
    "nextExecution": "2026-07-07 03:00:00"
  }
}
```

**注意事项**：
1. `invokeTarget`必须是白名单中的类和方法
2. Cron表达式格式必须正确
3. 任务创建后默认为启用状态

### 3. task_update - 修改任务

**描述**：修改现有定时任务的配置。

**必填参数**：
- `jobId`（integer）：任务ID

**可选参数**：
- `jobName`（string）：任务名称
- `jobGroup`（string）：任务组名
- `invokeTarget`（string）：调用目标字符串
- `cronExpression`（string）：Cron表达式
- `misfirePolicy`（string）：错过执行策略
- `concurrent`（string）：是否允许并发
- `remark`（string）：备注说明

**示例**：
```json
{
  "jobId": 5,
  "cronExpression": "0 0 4 * * ?",
  "remark": "调整为凌晨4点执行"
}
```

### 4. task_delete - 删除任务

**描述**：删除一个或多个定时任务。

**参数**：
- `jobIds`（array of integer）：要删除的任务ID数组

**示例**：
```json

  "jobIds": [5, 6, 7]
}
```

**返回示例**：
```json
{
  "success": true,
  "message": "成功删除 3 个任务"
}
```

### 5. task_run - 立即执行任务

**描述**：立即执行指定的定时任务（提前执行），不影响原定时计划。

**参数**：
- `jobId`（integer）：任务ID

**示例**：
```json
{
  "jobId": 5
}
```

**返回示例**：
```json
{
  "success": true,
  "message": "任务已触发执行: 每日备份任务"
}
```

**使用场景**：
- 测试任务是否正常运行
- 在计划时间之前手动触发执行
- 应急情况需要立即执行某任务

## Cron表达式详解

Cron表达式格式：`秒 分 时 日 月 星期 [年]`

### 字段说明

| 字段 | 允许值 | 允许特殊字符 |
|------|--------|--------------|
| 秒 | 0-59 | `, - * /` |
| 分 | 0-59 | `, - * /` |
| 时 | 0-23 | `, - * /` |
| 日 | 1-31 | `, - * ? / L W` |
| 月 | 1-12 或 JAN-DEC | `, - * /` |
| 星期 | 1-7 或 SUN-SAT | `, - * ? / L #` |
| 年 | 1970-2099 | `, - * /` |

### 特殊字符含义

- `*`：所有值（如：每分钟、每小时）
- `?`：不指定值（日和星期必须有一个用?）
- `-`：范围（如：`10-12`表示10点、11点、12点）
- `,`：列举（如：`MON,WED,FRI`表示周一、周三、周五）
- `/`：增量（如：`0/15`表示每15分钟）
- `L`：最后（如：日字段的`L`表示当月最后一天）
- `W`：工作日（如：`15W`表示离15号最近的工作日）
- `#`：第几个（如：`6#3`表示第3个星期五）

### 常用示例

```
每分钟                    0 * * * * ?
每5分钟                   0 0/5 * * * ?
每10分钟                  0 0/10 * * * ?
每30分钟                  0 0/30 * * * ?
每小时                    0 0 * * * ?
每天凌晨2点               0 0 2 * * ?
每天早上8点30分           0 30 8 * * ?
每周一9点                 0 0 9 ? * MON
每周一到周五9点           0 0 9 ? * MON-FRI
每月1号凌晨               0 0 0 1 * ?
每月最后一天23点59分      0 59 23 L * ?
每季度第一天凌晨          0 0 0 1 1,4,7,10 ?
每年1月1号凌晨            0 0 0 1 1 ?
```

## 安全说明

1. **审批机制**：定时任务插件默认需要审批，确保任务创建和修改操作需要人工确认。

2. **白名单限制**：`invokeTarget`必须在系统白名单中，防止执行恶意代码。

3. **权限控制**：只有授权的Agent才能调用此插件。

4. **执行日志**：所有任务执行都会记录日志，便于审计和追溯。

## 常见问题

### Q1：为什么我的任务创建失败？
**A**：可能的原因：
1. Cron表达式格式不正确
2. `invokeTarget`不在白名单中
3. 任务名称过长或包含特殊字符

### Q2：如何调试定时任务？
**A**：
1. 使用`task_run`立即执行任务
2. 查看定时任务执行日志（`/monitor/job/log`）
3. 使用`task_detail`查看任务配置

### Q3：任务误执行了怎么办？
**A**：
1. 使用`task_status`立即暂停任务
2. 修改Cron表达式调整执行时间
3. 如果不再需要，使用`task_delete`删除任务

### Q4：如何设置任务只执行一次？
**A**：
1. 创建任务后立即使用`task_run`执行
2. 执行完成后使用`task_status`暂停或`task_delete`删除
