## 📖 使用说明

### **界面结构**

打开【插件管理】菜单，你会看到两个Tab：

1. **插件管理** - 原有的插件管理功能
2. **MCP包扫描** - 新增的MCP包扫描功能

### **使用流程**

#### **场景: 配置已安装的MCP包**

**第1步**: 安装MCP包
```bash
pip install mcp-server-sqlite
```

**第2步**: 打开【插件管理】→ 切换到【MCP包扫描】Tab

**第3步**: 点击【扫描MCP包】按钮
- 系统会自动扫描Python和Node.js环境中的MCP包
- 显示统计信息（总计、已配置、未配置）

**第4步**: 查看扫描结果
- 切换到【Python包】或【Node.js包】子Tab
- 找到你安装的包（如 `mcp-server-sqlite`）

**第5步**: 配置插件
- 点击包列表中的【配置】按钮
- 查看自动填充的配置信息
- 点击【确认配置】

**第6步**: 查看和使用
- 切换回【插件管理】Tab
- 找到新配置的插件
- 可以启动、停止、同步工具等操作

## 🔍 关键代码说明

### **后端接口**

```java
// 扫描所有MCP包
GET /plugin/list/scanMcpPackages

// 自动配置MCP包为插件
POST /plugin/list/autoConfigureMcp
{
  "packageName": "mcp-server-sqlite",
  "runtimeType": "uvx"
}
```

### **前端API**

```javascript
// 扫描MCP包
const response = await scanMcpPackages()

// 自动配置
await autoConfigureMcp({

})
```

## 📝 注意事项

### **1. 环境要求**

- Python环境（用于扫描Python包）
- Node.js环境（用于扫描Node.js包）
- 包必须全局安装才能被扫描

### **2. 全局安装**

```bash
# ✅ 正确 - 全局安装
pip install mcp-server-sqlite
npm install -g @modelcontextprotocol/server-filesystem

# ❌ 错误 - 用户级或项目级安装
pip install --user mcp-server-sqlite
npm install @modelcontextprotocol/server-filesystem
```

### **3. 首次使用**

- 首次切换到"MCP包扫描"Tab会自动触发扫描
- 如果环境不可用，会显示警告提示
- 扫描超时时间为30秒

## 🐛 常见问题

### **Q1: 扫描不到包？**

**原因**: 
- Python/Node.js未安装
- 包未全局安装
- pip/npm不在PATH中

**解决**:
```bash
# 检查环境
python --version
pip --version
node --version
npm --version

# 确保全局安装
pip install mcp-server-sqlite
npm install -g package-name
```

### **Q2: 配置后找不到插件？**

**解决**: 切换回"插件管理"Tab，查看插件列表

### **Q3: 扫描慢或超时？**

**原因**: pip/npm命令执行慢

**解决**: 
- 配置镜像源
- 检查网络连接
- 手动执行 `pip list` 和 `npm list` 测试
