# MCP插件管理流程图

## 1. 系统架构

```

│                        前端界面                              │

└─────────────────────────────────────────────────────────────┘

│  /plugin/list/scanMcpPackages - 扫描MCP包（不写数据库）      │

## 2. MCP包扫描流程

```
┌──────────────────────────────────────────────────────────────┐
│ 用户操作：点击"扫描MCP包"按钮                                  │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 前端：调用 GET /plugin/list/scanMcpPackages                   │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 后端：扫描Python包                                             │
│  1. 执行命令：cmd /c python -m pip list --format=json        │
│  2. 解析JSON输出                                              │
│  3. 筛选MCP相关包（包名包含mcp-server/mcp-/@modelcontextprotocol）│
│  4. 查询ai_plugin表，检查每个包是否已配置                      │
│  5. 返回包列表，标记已配置/未配置状态                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 后端：扫描Node.js包                                            │
│  1. 执行命令：cmd /c npm list -g --depth=0 --json            │
│  2. 解析JSON输出                                              │
│  3. 筛选MCP相关包                                             │
│  4. 查询ai_plugin表，检查每个包是否已配置                      │
│  5. 返回包列表，标记已配置/未配置状态                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 后端：合并结果                                                 │
│  {                                                            │
│    "pythonPackages": [...],  // Python MCP包列表              │
│    "nodePackages": [...],    // Node.js MCP包列表             │
│    "pythonAvailable": true,  // Python环境是否可用            │
│    "nodeAvailable": true,    // Node.js环境是否可用           │
│    "totalCount": 10,         // 总包数                        │
│    "configuredCount": 3,     // 已配置数                      │
│    "unconfiguredCount": 7,   // 未配置数                      │
│    "scanTime": "2026-06-30 10:30:00"  // 扫描时间             │
│  }                                                            │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 前端：显示扫描结果                                             │
│  - 统计卡片：总计/已配置/未配置                                │
│  - Python包表格：显示包名、版本、分类、状态、操作按钮          │
│  - Node.js包表格：显示包名、版本、分类、状态、操作按钮         │
│  - 已配置的包：显示"已配置"标签，按钮为"查看"                  │
│  - 未配置的包：显示"未配置"标签，按钮为"配置"                  │
└──────────────────────────────────────────────────────────────┘
```

**重要**：扫描阶段**不会写入数据库**，只是读取系统已安装的包并显示状态。

## 3. MCP包配置流程

```
┌──────────────────────────────────────────────────────────────┐
│ 用户操作：在扫描结果中点击"配置"按钮                            │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 前端：打开配置对话框                                           │
│  预填充信息：                                                  │
│  - 包名：mcp-server-sqlite（不可编辑）                         │
│  - 插件名称：SQLite Database（可编辑）                        │
│  - 运行时：uvx（不可编辑）                                     │
│  - 分类：数据库（可选择）                                      │
│  - 环境变量：{}（可编辑，JSON格式）⭐                          │
│  - 启动命令：uvx mcp-server-sqlite（可编辑）                  │
│  - 需要审批：是（可选择，数据库/文件系统默认需要）              │
│  - 状态：禁用（可选择，默认禁用）⭐                             │
│  - 描述：（可编辑）                                            │

                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 用户操作：填写配置信息                                         │
│  例如配置SQLite数据库插件：                                    │
│  - 环境变量：{"DB_PATH":"/data/app.db","DB_READONLY":"false"} │
│  - 启动命令：uvx mcp-server-sqlite --db-path /data/app.db     │
│  - 需要审批：是                                                │
│  - 状态：禁用（先配置，稍后测试通过再启用）                     │

└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐

│  {                                                            │
│    "packageName": "mcp-server-sqlite",                       │
│    "runtimeType": "uvx",                                     │

│    "category": "database",                                   │
│    "envVars": "{\"DB_PATH\":\"/data/app.db\"}",              │

│  }                                                            │

                              ↓
┌──────────────────────────────────────────────────────────────┐

                              ↓

## 4. 数据流对比

### 扫描阶段（只读，不写数据库）
```
系统已安装的包 → 命令行工具(pip/npm) → 包列表 → 过滤MCP包 → 查询数据库状态 → 显示
                                                    ↑
                                             ai_plugin表
                                          （只读，不修改）
```

### 配置阶段（写入数据库）
```
用户填写配置 → 前端验证 → 后端API → ai_plugin表（INSERT）→ 配置完成
```

## 5. 关键区别总结

| 操作 | 扫描MCP包 | 配置MCP插件 |
|------|----------|-----------|
| 数据来源 | 系统环境（pip/npm） | 用户输入 |
| 数据库操作 | **只读**（SELECT） | **写入**（INSERT） |
| 是否需要用户输入 | 否 | 是 |
| 环境变量 | 不涉及 | 用户配置（关键） |
| 默认状态 | 不适用 | 禁用（'1'） |
| 表操作 | 查询ai_plugin | 插入ai_plugin |

## 6. 统一表结构的优势

使用同一张 `ai_plugin` 表存储所有插件（内置和MCP）：

✅ **统一管理**：所有插件在同一个界面管理
✅ **统一权限**：所有插件使用相同的权限模型
✅ **统一审批**：所有插件可以使用相同的审批流程
✅ **代码复用**：插件管理代码可以复用
✅ **数据一致性**：避免多表同步问题

通过 `plugin_type` 字段区分：
- `'mcp'` = MCP插件（从系统扫描配置）
- `'builtin'` = 内置插件（手动创建）
