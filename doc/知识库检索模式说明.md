# 知识库检索模式说明

## 📚 概述

系统支持两种知识库检索模式，可以在 **Agent 权限管理 → 知识库授权** 中为每个知识库独立配置。

## 🔄 两种检索模式对比

| 特性 | 自动注入（auto） | 工具模式（tool） |
|------|-----------------|------------------|
| **触发时机** | 对话开始前自动检索 | Agent 主动决定何时检索 |
| **调用方式** | 系统自动注入到 system prompt | Agent 调用 `knowledge_search` 工具 |
| **Token 消耗** | 每次对话都消耗（无论是否需要） | 仅在需要时消耗 |
| **适用场景** | 问题总是需要知识库支持 | 问题有时需要知识库支持 |
| **灵活性** | 低 - 固定注入 | 高 - AI 智能决策 |
| **响应速度** | 快（一次对话） | 慢（需要额外的工具调用） |
| **成本** | 高（每次都检索） | 低（按需检索） |

## 🎯 模式一：自动注入（auto）

### 工作流程

```
用户提问
    ↓
[系统自动检索知识库]
    ↓
将检索结果注入到 system prompt
    ↓
AI 基于注入的知识回答
    ↓
返回结果（一次对话完成）
```

### 特点

✅ **优点**：
- 响应快速，一次对话即可得到答案
- 无需 AI 理解何时需要检索
- 适合专业客服、文档助手等场景

❌ **缺点**：
- 每次都检索，Token 消耗大
- 可能注入无关内容
- 无法根据对话上下文动态调整

### 适用场景

1. **专业客服机器人**
   - 用户问题总是需要查询知识库
   - 如：产品咨询、售后支持

2. **文档助手**
   - 所有问题都基于特定文档
   - 如：操作手册问答、规章制度查询

3. **合规检查**
   - 需要引用准确的政策法规
   - 如：财务审核、法律咨询

### 配置示例

```yaml
Agent: 产品客服助手
知识库: 产品操作手册
检索模式: auto（自动注入）
元数据过滤: {"doc_category": "产品文档"}
```

## 🛠️ 模式二：工具模式（tool）

### 工作流程

```
用户提问
    ↓
AI 分析是否需要检索知识库
    ↓
[AI 决定] 需要 → 调用 knowledge_search 工具
         不需要 → 直接回答
    ↓
获取检索结果
    ↓
AI 基于结果回答
    ↓
返回结果（可能有多次对话）
```

### 特点

✅ **优点**：
- 按需检索，节省 Token
- AI 可根据上下文决定检索时机
- 可以多次检索不同关键词
- 更智能、更灵活

❌ **缺点**：
- 响应较慢（多轮对话）
- 依赖 AI 判断能力
- 可能遗漏检索

### 适用场景

1. **通用助手**
   - 问题多样化，不是所有问题都需要知识库
   - 如：企业内部助手、多功能机器人

2. **混合场景**
   - 既有闲聊，又有专业咨询
   - 如：智能客服（先闲聊再业务）

3. **复杂查询**
   - 需要多次检索、逐步细化
   - 如：研究助手、技术支持

### 工具调用示例

```json
// AI 自动生成的工具调用
{
  "tool": "knowledge_search",
  "arguments": {
    "query": "如何重置密码",
    "top_k": 3,
    "min_score": 0.6
  }
}
```

### 配置示例

```yaml
Agent: 企业智能助手
知识库: 内部规章制度
检索模式: tool（工具模式）
元数据过滤: {"doc_category": "人事制度"}
```

## ⚙️ 如何配置

### 步骤 1：进入 Agent 权限管理

1. 打开 **Agent 管理** 页面
2. 点击要配置的 Agent 的 **权限** 按钮
3. 切换到 **知识库授权** 标签页

### 步骤 2：选择知识库

- 勾选要授权的知识库
- 每个知识库可以独立配置检索模式

### 步骤 3：配置检索模式

在 **检索模式** 列中选择：
- **自动注入**：每次对话前自动检索
- **工具模式**：AI 主动调用工具检索

### 步骤 4：（可选）配置元数据过滤

在 **元数据过滤** 列中输入 JSON 格式的过滤条件：

```json
{"doc_category": "财务", "doc_tags": "政策"}
```

### 步骤 5：保存配置

点击 **保存** 按钮使配置生效。

## 🔍 元数据过滤说明

### 作用

限制检索范围，只在符合条件的文档中搜索。

### 格式

```json
{
  "字段名1": "值1",
  "字段名2": "值2"
}
```

### 支持的字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `doc_category` | 文档分类 | "财务", "人事", "技术" |
| `doc_tags` | 文档标签 | "政策", "流程", "规范" |
| `document_id` | 文档 ID | "123" |
| `chunk_index` | 分片索引 | "0" |

### 示例

```json
// 只在财务类文档中搜索
{"doc_category": "财务"}

// 搜索带有"政策"标签的技术文档
{"doc_category": "技术", "doc_tags": "政策"}

// 只搜索特定文档
{"document_id": "456"}
```

## 💡 最佳实践

### 1. 根据使用频率选择模式

- **高频场景** → 使用 `auto`（减少延迟）
- **低频场景** → 使用 `tool`（节省成本）

### 2. 混合使用两种模式

一个 Agent 可以同时授权多个知识库，为不同知识库设置不同模式：

```yaml
Agent: 企业助手
知识库配置:
  - 常见问题 FAQ: auto（自动注入）  # 高频
  - 技术文档: tool（工具模式）      # 低频
  - 规章制度: tool（工具模式）      # 按需
```

### 3. 使用元数据过滤提高精度

```json
// ✅ 好的做法：限定范围
{"doc_category": "产品A", "doc_tags": "故障排查"}

// ❌ 不好的做法：范围太广
{}
```

### 4. 设置合理的相似度阈值

- **自动注入**：建议 `0.5` - `0.6`（稍宽松）
- **工具模式**：建议 `0.6` - `0.7`（稍严格）

### 5. 监控和优化

定期查看：
- 检索命中率
- Token 消耗
- 用户反馈

根据数据调整模式和参数。

## 🔧 技术细节

### 自动注入实现

文件：`KnowledgeContextInjector.java`

```java
// 在对话开始前自动执行
String kbContext = knowledgeContextInjector.inject(agentId, userMessage);
if (kbContext != null) {
    systemPrompt += kbContext;
}
```

### 工具模式实现

文件：`KnowledgeSearchPlugin.java`

```java
// 注册为内置插件
@Component
public class KnowledgeSearchPlugin implements IBuiltinPlugin {
    // 提供 knowledge_search 工具
}
```

### 数据库表结构

```sql
CREATE TABLE `ai_agent_knowledge` (
  `id`              bigint(20)   NOT NULL AUTO_INCREMENT,
  `agent_id`        bigint(20)   NOT NULL,
  `kb_id`           bigint(20)   NOT NULL,
  `retrieval_mode`  varchar(10)  DEFAULT 'auto',  -- ← 检索模式
  `metadata_filter` varchar(500) DEFAULT NULL,    -- ← 元数据过滤
  `status`          char(1)      DEFAULT '0',
  PRIMARY KEY (`id`)
);
```

## ❓ 常见问题

### Q1：可以同时使用两种模式吗？

**A**：可以！一个 Agent 可以授权多个知识库，每个知识库可以设置不同的模式。

### Q2：工具模式下，AI 会忘记检索吗？

**A**：可能。建议在 Agent 的系统提示词中明确告知 AI 有知识库工具可用：

```
你可以使用 knowledge_search 工具检索知识库中的专业资料。
当用户提出需要查询文档、规范、政策等问题时，请主动调用该工具。
```

### Q3：如何查看哪个模式被使用了？

**A**：
- **自动注入**：SSE 事件中会显示"已检索知识库相关资料"
- **工具模式**：工具调用日志中会记录 `knowledge_search` 调用

### Q4：可以动态切换模式吗？

**A**：可以！在 Agent 权限管理中修改检索模式后立即生效（新的对话会话）。

### Q5：两种模式的成本差异有多大？

**A**：假设一个检索返回 3 条结果，每条 500 tokens：
- **自动注入**：每次对话 +1500 tokens（无论是否需要）
- **工具模式**：仅在需要时 +1500 tokens + 工具调用开销（约 100 tokens）

如果只有 30% 的对话需要知识库，工具模式可节省约 60% 的成本。
