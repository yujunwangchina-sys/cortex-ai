# MCP插件配置使用指南

## 快速开始

### 步骤1：扫描系统中的MCP包

1. 打开"AI插件管理"页面
2. 点击"**MCP包扫描**"标签页
3. 点击"**扫描MCP包**"按钮
4. 等待扫描完成（约5-10秒）

**结果显示**：
- 统计卡片：显示总计、已配置、未配置的包数量
- Python包列表：显示Python环境中安装的MCP包
- Node.js包列表：显示Node.js环境中安装的MCP包

**状态说明**：
- 🟢 **已配置**：该包已经在插件管理中配置，可以点击"查看"跳转
- 🟡 **未配置**：该包尚未配置，可以点击"配置"进行配置

---

### 步骤2：配置MCP插件

1. 在扫描结果中找到未配置的包
2. 点击"**配置**"按钮
3. 填写配置信息：

#### 必填字段
- **包名**：自动填充，不可修改
- **插件名称**：给插件起一个友好的名字
- **运行时类型**：自动填充（uvx用于Python，npx用于Node.js）

#### 重要配置

##### 🔹 分类
选择插件的功能分类：
- **数据库**：SQLite、PostgreSQL、MySQL等数据库插件
- **文件系统**：文件访问、目录操作等插件
- **网络搜索**：天气查询、新闻搜索、网页抓取等
- **实用工具**：其他辅助工具
- **自定义**：自定义分类

##### 🔹 环境变量（⭐ 关键配置）
用于配置数据库密码、API密钥等敏感信息，必须是**JSON格式**。

**示例**：
```json
{
  "DB_HOST": "localhost",
  "DB_USER": "root",
  "DB_PASSWORD": "123456",
  "API_KEY": "your_api_key_here",
  "ALLOWED_PATHS": "/home/user/documents"
}
```

**常见场景**：
- 数据库插件：数据库连接信息（主机、用户、密码）
- API插件：API密钥、访问令牌
- 文件系统插件：允许访问的路径
- 网络插件：代理设置、超时配置

##### 🔹 启动命令
插件的启动命令，可以添加额外参数。

**示例**：
- Python包：`uvx mcp-server-sqlite --db-path /data/app.db`
- Node.js包：`npx -y @modelcontextprotocol/server-filesystem`

##### 🔹 需要审批
是否需要管理员审批才能使用此插件。

**建议设置**：
- ✅ **数据库类插件**：需要审批（涉及数据访问）
- ✅ **文件系统类插件**：需要审批（涉及文件操作）
- ⚪ **网络搜索类插件**：不需要审批
- ⚪ **实用工具类插件**：根据功能决定

##### 🔹 状态
插件的启用状态。

**默认值**：🔴 **禁用**

**为什么默认禁用？**
- 安全考虑：新配置的插件需要测试
- 资源管理：避免自动启动不必要的插件
- 用户控制：先检查配置，确认无误后再启用

4. 点击"**确认配置**"按钮
5. 看到"配置成功！插件已添加到插件列表"提示

---

### 步骤3：启用插件

1. 切换到"**插件管理**"标签页
2. 找到刚才配置的插件（状态为"禁用"）
3. 启用方式任选一种：
   - **方式1**：点击状态开关，确认启用
   - **方式2**：点击"修改"按钮，将状态改为"启用"，保存

4. 插件状态变为"启用"（绿色），系统可以使用该插件

## 配置示例

### 示例1：SQLite数据库插件

**场景**：配置一个SQLite数据库访问插件

```
包名：mcp-server-sqlite
插件名称：SQLite数据库
分类：数据库
环境变量：
{
  "DB_PATH": "/data/myapp.db",
  "DB_READONLY": "false"
}
启动命令：uvx mcp-server-sqlite --db-path /data/myapp.db
需要审批：是（推荐）
状态：禁用（先配置，测试后启用）
描述：提供SQLite数据库查询和操作功能
```

## 常见问题

### Q1: 扫描不到MCP包？
**A**: 检查以下几点：
1. 确认已安装Python/Node.js环境
2. 确认已安装MCP包：
   - Python: `pip install mcp-server-xxx`
   - Node.js: `npm install -g @modelcontextprotocol/server-xxx`
3. 确认环境变量PATH配置正确
4. 重启应用后重新扫描

### Q2: 环境变量格式错误？
**A**: 环境变量必须是有效的JSON格式：
- ✅ 正确：`{"KEY":"value"}`
- ❌ 错误：`{KEY:value}` 或 `'key':'value'`
- 使用双引号，不要使用单引号
- 键和值都要加引号

### Q3: 配置后在哪里查看插件？
**A**: 切换到"插件管理"标签页，可以看到所有已配置的插件（包括内置插件和MCP插件）

### Q4: 如何修改已配置的插件？
**A**: 
1. 在"插件管理"中找到该插件
2. 点击"修改"按钮
3. 修改配置（包括环境变量）
4. 保存

### Q5: 配置后插件是禁用状态，怎么办？
**A**: 这是正常的！默认禁用是为了安全。检查配置无误后，手动启用插件即可。

### Q6: 同一个包可以配置多次吗？
**A**: 不可以。系统会检测包名（package_name），如果已经配置过，会提示"插件已存在"。

### Q7: 内置插件和MCP插件有什么区别？
**A**: 
- **内置插件**：手动创建，通过Java类实现
- **MCP插件**：从系统扫描配置，通过命令行启动
- 两者在"插件管理"中统一管理，权限和审批机制相同

### Q8: 环境变量保存在哪里？安全吗？
**A**: 
- 保存在数据库的 `ai_plugin.env_vars` 字段（TEXT类型）
- 建议后续版本实现加密存储
- 当前版本请注意数据库访问权限控制

---

## 安全建议

1. **数据库和文件系统插件务必启用审批**
2. **环境变量中的敏感信息定期更新**
3. **不使用的插件及时禁用或删除**
4. **定期审查已启用的插件列表**
5. **限制插件访问范围**（如文件路径、数据库权限）

## 附录：MCP包安装方法

### Python MCP包
```bash
# 安装SQLite插件
pip install mcp-server-sqlite

# 安装文件系统插件
pip install mcp-server-filesystem

# 安装其他MCP插件
pip install mcp-server-xxx
```

### Node.js MCP包
```bash
# 安装文件系统插件
npm install -g @modelcontextprotocol/server-filesystem

# 安装其他MCP插件
npm install -g @modelcontextprotocol/server-xxx
```

安装后，刷新扫描即可看到新安装的包。
