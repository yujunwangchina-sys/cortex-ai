# MCP插件管理系统 - 测试指南

## 📋 系统概述

MCP插件管理系统已完成开发，包含以下功能模块：

### ✅ 已完成功能

1. **插件管理** - 完整的MCP插件CRUD操作
2. **运行时环境检测** - Python/Node.js/uvx/npx环境检测
3. **包管理** - Python和Node.js包的安装/卸载/列表查看
4. **执行日志** - 插件工具调用日志记录和查询
5. **预加载机制** - 系统启动时自动加载所有启用的MCP插件
6. **定时任务** - 每5分钟清理空闲会话

### 🏗️ 系统架构

```
系统启动 → PluginConfig预加载 → 启动所有MCP进程 → McpProcessManager（全局进程池）
                                                              ↓
Agent调用工具 → McpSessionManager.getClient() → McpClient（JSON-RPC 2.0客户端）
                                                              ↓
                                                    复用进程池中的进程
```

## 🔧 常见问题排查

### Q1: 菜单不显示怎么办？

**原因**：SQL脚本未执行或权限未分配

**解决**：
1. 确认已执行`ai_plugin_system.sql`
2. 登录管理员账号
3. 进入【系统管理】→【菜单管理】，检查"插件管理"菜单是否存在
4. 进入【系统管理】→【角色管理】，给当前角色分配插件管理权限

### Q2: MCP插件启动失败

**错误日志**：
```
❌ MCP插件启动失败: mcp-filesystem - Could not execute [npx...]
```

**原因**：系统缺少对应的运行时环境

**解决**：
1. 访问【运行时环境】页面，检查环境状态
2. 根据"安装指南"安装缺失的运行时
3. 推荐安装：
   - `pip install uv`（Python包管理器，提供uvx）
   - Node.js自带npx（安装Node.js即可）

### Q3: 工具同步失败

**错误提示**：`插件进程未启动或已停止`

**原因**：MCP进程未成功启动

**解决**：
1. 检查运行时环境是否齐全
2. 查看后端启动日志，确认预加载是否成功
3. 尝试在【插件列表】页面重启插件：
   - 先禁用插件
   - 再启用插件
   - 系统会自动重启进程

### Q4: 包管理页面显示空列表

**原因**：Python/Node.js未安装或命令不在PATH中

**解决**：
1. 确认运行时已正确安装
2. 检查环境变量PATH是否包含python/node路径
3. 重启后端服务（环境变量变更需重启）

### Q5: 前端路由404

**原因**：前端代码未编译或路由未注册

**解决**：
1. 确认前端代码已存在：`e:\java\Cortex-Vue\Cortex-Vue3\src\views\plugin\`
2. 重新编译前端：`npm run dev`
3. 清除浏览器缓存，强制刷新（Ctrl+F5）

## 📊 数据库表说明

### 1. ai_plugin（插件表）

主要字段：
- `plugin_code`：插件唯一编码（如mcp-sqlite）
- `plugin_type`：插件类型（mcp/builtin）
- `runtime_type`：运行时（uvx/npx/venv/npm）
- `package_name`：包名
- `start_command`：启动命令（JSON数组）
- `status`：状态（0启用/1禁用）

### 2. ai_plugin_tool（工具表）

主要字段：
- `plugin_id`：关联插件ID
- `tool_name`：工具名称
- `tool_code`：工具编码
- `input_schema`：输入参数schema（JSON Schema）
- `example_input`：示例输入

### 3. ai_plugin_execution_log（执行日志表）

主要字段：
- `session_id`：会话ID（支持会话隔离）
- `plugin_code`：插件名称
- `tool_name`：工具名称
- `input_params`：输入参数（JSON）
- `output_result`：输出结果（JSON）
- `execution_time`：执行时长（毫秒）
- `status`：状态（0成功/1失败）
- `error_message`：错误信息

### 4. ai_plugin_process（进程状态表）

主要字段：
- `plugin_code`：插件名称
- `pid`：操作系统进程ID
- `status`：状态（starting/running/stopped/error）
- `last_heartbeat`：最后心跳时间
