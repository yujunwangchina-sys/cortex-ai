# 版本号保存修复说明

## 问题描述
在"加载MCP包"功能中，扫描到的MCP包版本号未被保存到`ai_plugin`表中。

## 问题原因
前端在调用`enableMcpPackage`接口时已经传递了`version`参数，但是后端Controller没有将版本号传递给Service层。

## 修复内容

### 1. 修改AiPluginController.java
**文件位置**: `ruoyi-admin\src\main\java\com\ruoyi\web\controller\plugin\AiPluginController.java`

**修改内容**:
```java
// 修改前
Long pluginId = mcpPackageScanService.enableMcpPackage(
    packageInfo.getPackageName(), 
    packageInfo.getRuntimeType(),
    packageInfo.getEnvVars(),
    packageInfo.getRequireApproval()
);

// 修改后
Long pluginId = mcpPackageScanService.enableMcpPackage(
    packageInfo.getPackageName(), 
    packageInfo.getRuntimeType(),
    packageInfo.getVersion(),  // 新增版本号参数
    packageInfo.getEnvVars(),
    packageInfo.getRequireApproval()
);
```

### 2. 修复McpPackageScanServiceImpl.java中的重复日志
**文件位置**: `ruoyi-system\src\main\java\com\ruoyi\plugin\service\impl\McpPackageScanServiceImpl.java`

**修改内容**: 删除了`enableMcpPackage`方法中重复的日志语句。

## 数据流程

1. **前端扫描** → `McpLoader.vue`调用`scanMcpPackages`接口获取本地MCP包列表
2. **显示版本** → 在弹窗中显示每个包的版本号（来自`packageInfo.version`）
3. **加载包** → 用户点击"加载"按钮，调用`enableMcpPackage`接口，传递完整的`packageInfo`对象（包含版本号）
4. **Controller层** → `AiPluginController.enableMcpPackage`接收`McpPackageInfo`对象，提取`version`字段
5. **Service层** → `McpPackageScanServiceImpl.enableMcpPackage`接收版本号参数
6. **数据库保存** → 创建或更新`ai_plugin`记录时，设置`version`字段

## 测试步骤

1. 重新编译后端（需要配置JDK环境）:
   ```bash
   mvn clean compile -DskipTests
   ```

2. 重启后端服务

3. 前端操作:
   - 进入插件管理页面
   - 点击"加载MCP包"按钮
   - 在弹窗中查看MCP包列表（应显示版本号）
   - 点击某个包的"加载"按钮
   - 关闭弹窗，查看插件列表中该包的版本号列是否已填充

## 预期结果
- 插件列表的"版本"列应该显示从扫描中获取的版本号
- 数据库`ai_plugin`表的`version`字段应该正确保存版本号
