# 插件分类系统 - 快速开始指南

## 🚀 快速部署

### **步骤1: 编译后端**

```bash
cd e:\java\Cortex-Vue
mvn clean compile -pl cortex-system,cortex-admin
```

### **步骤2: 重启后端服务**

```bash
# 如果使用IDEA，直接点击重启按钮
# 或使用命令行
mvn spring-boot:run -pl cortex-admin
```

### **步骤3: 更新前端页面**

**选项A: 替换原页面（推荐）**

```bash
cd e:\java\Cortex-Vue\Cortex-Vue3\src\views\plugin\list

# 备份原文件
copy index.vue index.vue.backup

# 使用新版本
copy indexWithTabs.vue index.vue
```

**选项B: 保留原页面，添加新路由**

手动在路由配置中添加新页面即可。

### **步骤4: 重启前端**

```bash
cd e:\java\Cortex-Vue\Cortex-Vue3
npm run dev
```

## 📝 功能演示

### **场景: 配置SQLite数据库插件**

#### **第1步: 安装MCP包**

```bash
pip install mcp-server-sqlite
```

#### **第2步: 扫描包**

1. 打开浏览器，访问系统
2. 进入【插件管理】菜单
3. 点击【MCP插件】Tab
4. 点击【扫描MCP包】按钮

#### **第3步: 配置插件**

1. 在Python包列表中找到`mcp-server-sqlite`
2. 查看包信息：
   - 包名: mcp-server-sqlite
   - 版本: 1.0.0（示例）
   - 分类: 数据库
   - 状态: 未配置
3. 点击【配置】按钮
4. 系统自动填充配置信息
5. 点击【确认配置】

#### **第4步: 启动使用**

1. 切换到【内置插件】Tab
2. 找到新配置的"SQLite数据库"插件
3. 点击【启动】按钮
4. 点击【同步】按钮同步工具
5. 点击【工具】按钮查看可用工具

## 🎯 核心功能

### **1. MCP插件Tab**

**功能**:
- ✅ 自动扫描已安装的Python/Node.js MCP包
- ✅ 显示统计信息（总计、已配置、未配置）
- ✅ 区分Python包和Node.js包
- ✅ 显示包的详细信息（版本、分类、描述）
- ✅ 一键配置为插件
- ✅ 查看包元数据

**操作按钮**:
- 【扫描MCP包】: 扫描系统已安装的MCP相关包
- 【刷新列表】: 重新扫描并更新列表
- 【配置】: 将未配置的包自动配置为插件
- 【查看】: 查看已配置插件的详情
- 【详情】: 查看包的完整元数据

### **2. 内置插件Tab**

**功能**:
- ✅ 显示所有已配置的插件（包括MCP和内置）
- ✅ 插件的CRUD操作
- ✅ 启动/停止MCP插件
- ✅ 同步插件工具
- ✅ 查看插件工具列表
- ✅ 状态管理（启用/禁用）

**操作按钮**:
- 【新增插件】: 手动创建新插件
- 【修改】: 编辑插件配置
- 【删除】: 删除插件
- 【启动】: 启动MCP插件进程
- 【停止】: 停止MCP插件进程
- 【同步】: 同步插件的工具列表
- 【工具】: 查看插件提供的工具
- 【重新加载全部】: 重启所有已启用的MCP插件

## 🔍 扫描逻辑说明

### **Python包扫描**

**命令**: `pip list`

**筛选规则**:
- 包名包含 `mcp-server`
- 包名包含 `mcp-`
- 包名包含 `@modelcontextprotocol`

**示例输出**:
```
mcp-server-sqlite==1.0.0
mcp-server-postgres==2.1.0
```

### **Node.js包扫描**

**命令**: `npm list -g --depth=0 --json`

**筛选规则**:
同Python包

**示例输出**:
```json
{
  "dependencies": {
    "@modelcontextprotocol/server-filesystem": {
      "version": "1.0.0"
    }
  }
}
```

## ⚙️ 自动配置说明

### **配置流程**

1. **用户点击"配置"按钮**

2. **系统自动生成插件名称**
   - 规则: 移除前缀`@modelcontextprotocol/server-`或`mcp-server-`
   - 添加`mcp-`前缀（如果没有）
   - 示例: `@modelcontextprotocol/server-filesystem` → `mcp-filesystem`

3. **填充元数据**
   - 从内置的官方包列表中查找
   - 如果找到，使用官方元数据
   - 如果未找到，使用默认值

4. **生成启动命令**
   - uvx包: `["uvx", "package-name"]`
   - npx包: `["npx", "-y", "package-name"]`

5. **插入数据库**
   - 表: `ai_plugin`
   - 状态: 默认启用（status='0'）

### **官方包列表**

系统内置以下官方包的完整元数据：

| 包名 | 显示名称 | 分类 |
|------|---------|------|
| mcp-server-sqlite | SQLite数据库 | database |
| mcp-server-postgres | PostgreSQL数据库 | database |
| @modelcontextprotocol/server-filesystem | 文件系统 | file_system |
| @modelcontextprotocol/server-git | Git版本控制 | utility |
| @modelcontextprotocol/server-brave-search | Brave搜索 | web_search |

## 🛠️ API接口

### **后端接口**

```http
# 扫描所有MCP包
GET /plugin/list/scanMcpPackages

# 扫描Python包
GET /plugin/list/scanPythonPackages

# 扫描Node.js包
GET /plugin/list/scanNodePackages

# 自动配置MCP插件
POST /plugin/list/autoConfigureMcp
Content-Type: application/json
{
  "packageName": "mcp-server-sqlite",
  "runtimeType": "uvx"
}

# 获取包元数据
GET /plugin/list/getPackageMetadata/{packageName}/{runtimeType}
```

### **前端API**

```javascript
import { 
  scanMcpPackages, 
  scanPythonPackages, 
  scanNodePackages,
  autoConfigureMcp,
  getPackageMetadata 
} from '@/api/plugin/plugin'

// 扫描所有包
const result = await scanMcpPackages()

// 自动配置
await autoConfigureMcp({
  packageName: 'mcp-server-sqlite',
  runtimeType: 'uvx'
})
```

## 🐛 故障排查

### **问题1: 扫描不到包**

**症状**: 点击"扫描MCP包"后，列表为空

**原因**:
- Python/Node.js环境未安装
- 未全局安装MCP包
- pip/npm命令不在PATH中

**解决**:
```bash
# 检查Python环境
python --version
pip --version

# 检查Node.js环境
node --version
npm --version

# 全局安装包（重要！）
pip install mcp-server-sqlite        # ✅ 正确
pip install --user mcp-server-sqlite # ❌ 错误，无法扫描

npm install -g @modelcontextprotocol/server-filesystem  # ✅ 正确
npm install @modelcontextprotocol/server-filesystem     # ❌ 错误，无法扫描
```

### **问题2: 配置失败**

**症状**: 点击"确认配置"后报错

**原因**:
- 插件名称已存在
- 必填字段缺失
- 数据库连接失败

**解决**:
- 查看后端日志
- 检查ai_plugin表是否已存在同名记录
- 尝试手动删除重复记录

### **问题3: 扫描超时**

**症状**: 扫描长时间无响应

**原因**:
- pip/npm命令执行慢
- 网络问题

**解决**:
- 增加超时时间（已设置30秒）
- 检查pip/npm镜像源配置
- 手动执行`pip list`和`npm list`测试

## 📝 注意事项

### **1. 全局安装**

MCP包必须全局安装才能被扫描到：

```bash
# ✅ 正确
pip install mcp-server-sqlite
npm install -g @modelcontextprotocol/server-filesystem

# ❌ 错误（用户级或项目级安装）
pip install --user mcp-server-sqlite
npm install @modelcontextprotocol/server-filesystem
```

### **2. 环境变量**

确保pip和npm在系统PATH中：

```bash
# Windows
where pip
where npm

# Linux/Mac
which pip
which npm
```

### **3. 权限问题**

某些系统可能需要管理员权限：

```bash
# Windows (以管理员运行CMD)
pip install mcp-server-sqlite

# Linux/Mac
sudo pip install mcp-server-sqlite
sudo npm install -g package-name
```

### **4. 重复配置**

- 同一个包只能配置一次
- 如需重新配置，先删除原插件记录
- 系统会自动检测并提示
