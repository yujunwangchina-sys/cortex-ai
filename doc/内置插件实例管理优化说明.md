# 内置插件实例管理优化说明

**实现日期**: 2026-07-01  
**功能描述**: 内置插件支持多实例管理，显示已有实例数量，与 Python 包管理对齐

---

## 一、背景

### 1.1 原有问题
- ❌ 内置插件状态只显示"未加载"，即使已经有实例也不显示
- ❌ 无法直观看到已经创建了多少个实例
- ❌ 与 Python/Node.js 包的"已有实例"状态不一致

### 1.2 优化目标
- ✅ 显示已加载的实例数量（如："3 个实例"）
- ✅ 支持重复加载，创建多个实例（类似 Python 包）
- ✅ 按钮文案根据状态变化："加载" → "再次加载"
- ✅ 与 Python/Node.js 包的行为保持一致

---

## 二、实现方案

### 2.1 数据模型变更

**BuiltinPluginInfo.java** 新增字段：
```java
/** 已加载的实例数量 */
private Integer instanceCount;
```

### 2.2 后端逻辑调整

**BuiltinPluginScanServiceImpl.java** - `scanBuiltinPlugins()` 方法：

**修改前**：
```java
// 检查是否已在插件列表中
AiPlugin queryParam = new AiPlugin();
queryParam.setBuiltinClass(plugin.getClass().getName());
List<AiPlugin> existingPlugins = aiPluginMapper.selectAiPluginList(queryParam);

if (existingPlugins != null && !existingPlugins.isEmpty()) {
    builtinInfo.setEnabled(true);
    builtinInfo.setPluginId(existingPlugins.get(0).getPluginId());
} else {
    builtinInfo.setEnabled(false);
}
```

**修改后**：
```java
// 检查是否已在插件列表中（统计实例数量）
AiPlugin queryParam = new AiPlugin();
queryParam.setBuiltinClass(plugin.getClass().getName());
List<AiPlugin> existingPlugins = aiPluginMapper.selectAiPluginList(queryParam);

if (existingPlugins != null && !existingPlugins.isEmpty()) {
    builtinInfo.setEnabled(true);
    builtinInfo.setPluginId(existingPlugins.get(0).getPluginId());
    builtinInfo.setInstanceCount(existingPlugins.size()); // 实例数量
} else {
    builtinInfo.setEnabled(false);
    builtinInfo.setInstanceCount(0);
}
```

**关键变化**：
- 不再只检查是否存在，而是统计同一 `builtinClass` 的所有插件记录
- 通过 `List.size()` 获取实例数量

## 三、前端界面优化

### 3.1 状态列显示

**McpLoader.vue** - 状态列：

**修改前**：
```vue
<el-table-column label="状态" align="center" width="100">
  <template #default="scope">
    <el-tag v-if="scope.row.loaded" type="success" size="small">已有实例</el-tag>
    <el-tag v-else type="info" size="small">未加载</el-tag>
  </template>
</el-table-column>
```

**修改后**：
```vue
<el-table-column label="状态" align="center" width="120">
  <template #default="scope">
    <el-tag v-if="scope.row.instanceCount > 0" type="success" size="small">
      {{ scope.row.instanceCount }} 个实例
    </el-tag>
    <el-tag v-else type="info" size="small">未加载</el-tag>
  </template>
</el-table-column>
```

**效果**：
- 未加载：灰色标签 "未加载"
- 已加载 1 个：绿色标签 "1 个实例"
- 已加载 3 个：绿色标签 "3 个实例"

### 3.2 操作按钮优化

**修改前**：
```vue
<el-button type="primary" size="small">
  加载
</el-button>
```

**修改后**：
```vue
<el-button type="primary" size="small">
  {{ scope.row.instanceCount > 0 ? '再次加载' : '加载' }}
</el-button>
```

**效果**：
- 首次加载：显示 "加载"
- 已有实例：显示 "再次加载"

### 3.3 加载逻辑优化

**handleLoadBuiltinPlugin** 函数：

**修改前**：
```javascript
function handleLoadBuiltinPlugin(pluginInfo) {
  if (pluginInfo.loaded) {
    // 提示用户输入名称
    ElMessageBox.prompt('该插件已有实例，请为新插件起一个名称', ...)
  } else {
    // 直接创建
    doLoadBuiltinPlugin(pluginInfo, null)
  }
}
```

**修改后**：
```javascript
function handleLoadBuiltinPlugin(pluginInfo) {
  if (pluginInfo.instanceCount > 0) {
    // 提示用户输入名称，显示当前实例数
    ElMessageBox.prompt(
      `该插件已有 ${pluginInfo.instanceCount} 个实例，请为新插件起一个名称（如：生产环境、测试环境）`, 
      ...
    )
  } else {
    // 首次加载，直接创建
    doLoadBuiltinPlugin(pluginInfo, null)
  }
}
```

**doLoadBuiltinPlugin** 函数：

**修改前**：
```javascript
loadBuiltinPlugin(data).then(() => {
  ElMessage.success(...)
  pluginInfo.loaded = true  // 标记为已加载
  ...
})
```

**修改后**：
```javascript
loadBuiltinPlugin(data).then(() => {
  ElMessage.success(...)
  pluginInfo.instanceCount = (pluginInfo.instanceCount || 0) + 1  // 实例数+1
  ...
})
```

### 3.4 扫描结果处理

**修改前**：
```javascript
builtinPlugins.value = (scanResult.value.builtinPlugins || []).map(plugin => ({
  ...plugin,
  loading: false,
  loaded: plugin.loaded || false
}))
```

**修改后**：
```javascript
builtinPlugins.value = (scanResult.value.builtinPlugins || []).map(plugin => ({
  ...plugin,
  loading: false,
  instanceCount: plugin.instanceCount || 0
}))
```

## 四、使用场景

### 4.1 场景1：首次加载插件

**操作流程**：
1. 打开"MCP插件加载"对话框
2. 切换到"内置插件"标签
3. 选择一个插件（如：skill-manager），显示"未加载"
4. 点击"加载"按钮
5. 插件加载到列表，状态变为"1 个实例"

### 4.2 场景2：创建多个实例

**操作流程**：
1. 已有一个 skill-manager 实例，显示"1 个实例"
2. 点击"再次加载"按钮
3. 弹出对话框：`该插件已有 1 个实例，请为新插件起一个名称`
4. 输入名称：`Skill管理器-测试环境`
5. 确认后创建新实例，状态变为"2 个实例"

### 4.3 场景3：查看实例列表

**在插件列表页面**：
```
插件名称              插件类型    状态
--------------------------------------
Skill管理器           内置       启用
Skill管理器-测试环境   内置       禁用
```

## 七、技术细节

### 7.1 数据库查询

// 统计同一 builtinClass 的插件数量
AiPlugin queryParam = new AiPlugin();

```javascript
// 加载成功后，本地更新实例数量，无需重新扫描

```
