# 插件扫描功能重构方案

## 一、需求

1. **统一扫描入口**：将"加载MCP包"改为"扫描插件"
2. **多Tab展示**：
   - Tab1: MCP Python包
   - Tab2: MCP Node.js包  
   - Tab3: 内置Java插件
3. **功能兼容**：
   - 测试连接支持内置插件
   - 同步工具支持内置插件
   - 启动/停止仅MCP插件支持
4. **配置管理**：内置插件配置存储到数据库，前端可编辑

## 二、数据库改造

### 1. 新增插件配置表
```sql
CREATE TABLE ai_plugin_config (
  config_id BIGINT PRIMARY KEY,
  plugin_id BIGINT NOT NULL,
  config_key VARCHAR(100) NOT NULL,
  config_value VARCHAR(500),
  config_type VARCHAR(20) DEFAULT 'string',
  is_required CHAR(1) DEFAULT '0',
  description VARCHAR(200),
  UNIQUE KEY (plugin_id, config_key)
);
```

## 三、后端改造

### 1. 新增服务
- `IPluginConfigService` - 插件配置管理
- `IBuiltinPluginScanService` - 内置插件扫描

### 2. 修改接口
- `McpPackageScanController` 改为 `PluginScanController`
- 新增 `/plugin/scan/builtin` 接口扫描内置插件

### 3. 内置插件改造
- 从`@Value`改为从数据库读取配置
- 添加`setPluginId()`方法
- 添加`getRequiredConfigs()`返回必需配置列表

## 四、前端改造

### 1. 组件重命名
- `McpLoader.vue` 改为 `PluginScanner.vue`
- 按钮文字从"加载MCP包"改为"扫描插件"

### 2. Tab结构
```vue
<el-tabs v-model="activeTab">
  <el-tab-pane label="Python MCP包" name="python">
    <!-- Python包列表 -->
  </el-tab-pane>
  <el-tab-pane label="Node.js MCP包" name="node">
    <!-- Node.js包列表 -->
  </el-tab-pane>
  <el-tab-pane label="内置插件" name="builtin">
    <!-- 内置插件列表 -->
  </el-tab-pane>
</el-tabs>
```

### 3. 配置编辑
内置插件列表增加"配置"按钮，打开配置对话框：
```vue
<el-dialog title="插件配置">
  <el-form-item label="API URL">
    <el-input v-model="config['dify.api-url']" />
  </el-form-item>
  <el-form-item label="API Key">
    <el-input v-model="config['dify.api-key']" type="password" />
  </el-form-item>
</el-dialog>
```

## 六、配置示例

### 内置插件配置
```json
{
  "dify.api-url": "http://dify-host/v1",
  "dify.api-key": "dataset-xxxxxxxx",
  "dify.dataset-id": "optional-id"
}
```

