# 插件管理功能说明

## 操作按钮详解

### 1. 🔗 测试连接（Test Connection）
**图标**: Connection  
**权限**: `plugin:list:test`  
**位置**: 每行插件的操作列

#### 功能说明
测试MCP插件的连接是否正常，验证插件是否能够成功启动并响应。

#### 工作流程
1. 临时启动MCP进程（不保存到进程池）
2. 创建临时测试客户端
3. 发送初始化请求（`initialize`）
4. 获取工具列表（`listTools`）
5. 返回工具数量和耗时
6. 关闭临时客户端和进程

#### 返回结果
- ✅ 成功: `连接成功！发现 X 个工具 (耗时: XXms)`
- ❌ 失败: `连接失败：错误信息 (耗时: XXms)`

#### 适用场景
- 新添加的插件，验证配置是否正确
- 修改了插件配置（命令、环境变量等），验证是否生效
- 排查插件无法使用的问题
- 检查插件依赖是否安装完整

#### 注意事项
- 每次测试都会重新启动进程，不影响已运行的插件
- 首次测试可能较慢（需要下载依赖）
- 测试失败可能原因：包名错误、环境变量缺失、网络问题、依赖未安装

---

### 2. ▶️ 启动（Start Plugin）
**图标**: VideoPlay  
**权限**: `plugin:list:edit`  
**位置**: 每行MCP插件的操作列  
**显示条件**: 仅MCP类型插件显示

#### 功能说明
启动MCP插件进程，使插件进入运行状态。

#### 工作流程
1. 根据插件配置构建启动命令
2. 设置环境变量
3. 使用`McpProcessManager`启动进程
4. 进程保存到全局进程池
5. 进程持续运行，直到手动停止或系统关闭

#### 适用场景
- 需要长期运行的插件服务
- 频繁调用的插件（避免每次都启动）
- 希望插件保持热状态

#### 与测试连接的区别
| 特性 | 测试连接 | 启动 |
|------|---------|------|
| 进程生命周期 | 临时，测试完即关闭 | 持久，保存到进程池 |
| 用途 | 验证配置 | 提供服务 |
| 输出重定向 | 不重定向stdout | 重定向到日志 |
| 进程管理 | 不纳入管理 | 统一管理 |

#### 注意事项
- 启动后进程会持续占用资源
- 多次启动同一个插件不会重复创建进程
- 修改插件配置后需要重新启动才能生效

---

### 3. ⏸️ 停止（Stop Plugin）
**图标**: VideoPause  
**权限**: `plugin:list:edit`  
**位置**: 每行MCP插件的操作列  
**显示条件**: 仅MCP类型插件显示

#### 功能说明
停止正在运行的MCP插件进程。

#### 工作流程
1. 从进程池中查找对应进程
2. 发送优雅关闭信号（`destroy`）
3. 等待5秒
4. 如果进程未退出，强制杀死（`destroyForcibly`）
5. 从进程池中移除

#### 适用场景
- 插件占用资源过多，需要释放
- 修改插件配置前，需要先停止
- 插件出现异常，需要重启
- 系统维护，停止所有插件

#### 注意事项
- 停止后所有与该插件的会话将失效
- 正在进行的工具调用可能失败
- 下次调用插件时需要重新启动

---

### 4. 🔄 同步工具（Sync Tools）
**图标**: Refresh  
**权限**: `plugin:tool:sync`  
**位置**: 每行插件的操作列

#### 功能说明
从MCP插件获取最新的工具列表，同步到数据库的`ai_plugin_tool`表。

#### 工作流程
1. 创建临时会话连接到插件
2. 调用`listTools`接口获取工具列表
3. 删除该插件的旧工具记录
4. 插入新的工具记录（包含工具名、描述、参数schema）
5. 关闭临时会话

#### 同步的内容
- **工具名称** (`name`): 工具的唯一标识
- **工具编码** (`code`): 同工具名称
- **描述** (`description`): 工具功能说明
- **输入参数** (`inputSchema`): JSON Schema格式的参数定义
- **状态**: 默认启用
- **排序**: 按索引顺序

#### 适用场景
- 插件更新后，新增了工具
- 插件工具的参数发生变化
- 工具列表与数据库不一致
- 初次配置插件后，获取工具列表

#### 返回结果
- ✅ 成功: `同步成功，共 X 个工具`
- ❌ 失败: `同步失败: 错误信息`

#### 注意事项
- 同步会删除旧工具，如果有自定义配置会丢失
- 同步前插件必须能够正常连接
- 如果插件未启动，会自动创建临时连接

---

### 5. 🔄 重新加载全部（Reload All）
**图标**: RefreshRight  
**权限**: `plugin:list:edit`  
**位置**: 顶部操作栏

#### 功能说明
批量重启所有启用状态的MCP插件。

#### 工作流程
1. 查询所有`status=0`（启用）且`type=mcp`的插件
2. 逐个执行：
   - 停止插件进程
   - 重新启动插件进程
3. 统计成功/失败数量
4. 返回汇总结果

#### 适用场景
- 系统升级后，需要重启所有插件
- 环境变量全局修改后
- 批量排查插件问题
- 系统维护后恢复服务

#### 返回结果
```
重新加载完成。总计: X, 成功: Y, 失败: Z
```

#### 注意事项
- 操作时间较长（取决于插件数量）
- 会中断所有正在使用的插件会话
- 失败的插件会在日志中记录详细错误

## 典型使用场景

### 场景1: 新增MCP插件
1. 点击"加载MCP包"或手动添加
2. 点击"测试连接"验证配置
3. 点击"同步工具"获取工具列表
4. （可选）点击"启动"使插件保持运行

### 场景2: 插件无法使用
1. 点击"测试连接"查看错误信息
2. 根据错误修改配置（环境变量、命令等）
3. 再次"测试连接"验证
4. 如果已启动，点击"停止"后重新"启动"

### 场景3: 插件更新
1. 点击"停止"停止旧版本
2. 更新插件包（通过包管理器）
3. 点击"测试连接"验证新版本
4. 点击"同步工具"更新工具列表
5. 点击"启动"启动新版本

### 场景4: 系统维护
1. 点击"重新加载全部"批量重启所有插件
2. 检查返回的成功/失败统计
3. 对失败的插件逐个排查

## 常见问题

### Q1: "测试连接"成功，但实际使用失败？
**A**: 测试连接只验证插件能否启动和响应，不保证工具能正确执行。需要检查：
- 环境变量是否配置完整（如API密钥）
- 工具需要的资源是否可访问（如数据库连接）
- 工具参数是否正确

### Q2: "启动"后还需要"测试连接"吗？
**A**: 不需要。"启动"已经成功就说明进程正常运行，可以直接调用工具。

### Q3: 什么时候需要"同步工具"？
**A**: 
- 插件首次配置后
- 插件更新到新版本后
- 工具列表为空或不完整时
- 插件官方文档说明新增了工具

### Q4: "停止"后数据会丢失吗？
**A**: 
- 进程状态会丢失（进程关闭）
- 数据库中的插件配置、工具列表保留
- 需要的数据（如数据库文件）取决于插件实现

### Q5: "重新加载全部"会影响正在使用的功能吗？
**A**: 会。所有进行中的工具调用可能失败，建议在维护时段执行。
